### 1. BERT 文本分类和实体识别有什么关系？分别使用什么 loss？

**它们的关系：** 在当前的 **JointBERT** 模型架构 (

BertForIntentClassificationAndSlotFilling) 中，文本分类（意图识别）和实体识别（槽位填充/NER）是**共享底层 BERT 编码器**的多任务学习。

- 它们共同利用同一个 BERT 提取出的深层语义特征。
- **文本分类（意图识别）** 使用了 BERT 输出的 `pooler_output`（对应 `[CLS]` 标记的特征表示，即 `bert_output[1]`），将其映射到句级别的类别空间中作分类。
- **实体识别（槽位填充）** 使用了 BERT 输出的 `token_output`（对应输入序列中所有 Token 的序列特征表示，即 `bert_output[0]`），将其映射到 Token 级别的类别空间作分类。

**使用的 Loss：** 根据 

main.py 第 19 行以及第 47-50 行的定义：

- **文本分类** 和 **实体识别** 分别使用的都是 **CrossEntropyLoss (交叉熵损失)**。
- `seq_loss = self.criterion(seq_output, seq_label_ids)` (文本意图分类交叉熵)
- `token_loss = self.criterion(active_logits, active_labels)` (过滤掉 mask 位置后的 Token 分类交叉熵)

### 2. 多任务训练 loss = seq_loss + token_loss 有什么坏处，如果存在训练不平衡的情况，如何处理？

**直接相加 (`loss = seq_loss + token_loss`) 的坏处：**

1. **Loss 量级和梯度差异（主导问题）：** `seq_loss` 是针对整句话单一标签进行计算的，而 `token_loss` 通常是对这句话中多个 Token 甚至所有 Token 进行计算然后取平均或求和。这容易导致两者在前向传播产生的 Loss 量级以及反向传播给共享 BERT 层带来的梯度幅度存在差异。其中一个任务（如实体识别由于样本量多）极易“掩盖”或主导了另一个任务的学习。
2. **收敛速度和学习难度不同（过拟合风险）：** 文本的意图分类相对简单，往往容易很快收敛；而序列的实体识别任务可能难度更大，需要更长的迭代次数。强行一比一相加，往往会导致意图识别任务已经过拟合甚至效果下降时，实体任务才刚刚收敛。

**训练不平衡的常见处理方法：**

1. **静态人工加权 (Weighted Sum)：** 最简单直接的处理方式：引入一个超参数 $\alpha$ 来重新分配两者的权重比，比如 `loss = alpha * seq_loss + (1 - alpha) * token_loss`，通过在验证集调参找到最佳分配。
2. **基以同方差不确定性的自适应加权 (Uncertainty Weighting)：** 为每个子任务的 Loss 增加一个可学习的参数（如 $\sigma_1, \sigma_2$），通过模型自己评估子任务的不确定度做自适应调整。 公式通常为：$Loss = \frac{L_{seq}}{2\sigma_1^2} + \log(\sigma_1) + \frac{L_{token}}{2\sigma_2^2} + \log(\sigma_2)$；这种方法能让模型自动下调那些具有高内部噪声的任务权重。
3. **动态任务优先梯度归一化算法 (GradNorm 等)：** 在反向传播时，统计各任务在共享层（Shared layers）产生的梯度范数（大小），动态惩罚梯度过大的任务，同时监控各任务 loss 的下降速率，迫使所有任务保持相近的训练速率。
4. **动态权重平均 (Dynamic Weight Averaging, DWA)：** 计算每个 epoch 中各项任务 Loss 下降的比例。如果某一任务 Loss 下降太快，就在下一个 epoch 中减少它相对应的 loss 权重，赋予难训练任务更多的关注。
5. **针对层/特定的学习率设置 (Task-Specific Learning Rate)：** 当共享层的参数更新不平衡时，可以给 `sequence_classification` 和 `token_classification` 上的最后映射层分别设置不同的学习率（比如通过 Adam 的 param_groups 设置），使它们能按照各自节奏下降。
