# BERT文本分类和实体识别的关系与多任务训练分析

## 一、文本分类和实体识别的关系

### 本质关系

两者都是分类任务，区别在于**分类的粒度不同**：

- **文本分类（意图识别）** 是句子级别的分类，用 BERT 输出的 `[CLS]` token 对应的向量（pooler_output）来预测整句话的意图类别，相当于"看完整句话，判断用户想干什么"。

- **实体识别（槽位填充/NER）** 是 token 级别的分类，用 BERT 输出的每一个 token 的向量来预测该位置属于哪个实体类别（B-xxx / I-xxx / O），相当于"逐字判断每个词属于哪种实体"。

两个任务**共享同一个 BERT 编码器**，只是在 BERT 输出后接了两个不同的分类头，一个用句子表示做分类，一个用每个 token 的表示做分类，这就是联合训练（Joint Training）的核心思路。

### 分别使用的 Loss

两个任务都使用**交叉熵损失（CrossEntropyLoss）**：

- 意图识别的 seq_loss：对整句话做多分类，计算预测意图和真实意图之间的交叉熵。

- 实体识别的 token_loss：对每个 token 做多分类，但需要**过滤掉 padding 位置**，只对真实 token 计算损失，避免 padding 干扰训练。

## 二、多任务训练直接相加的坏处

### 坏处一：两个 loss 的量级不一致

seq_loss 是一个句子对应一个标量，token_loss 是句子中所有 token 的平均，两者天然量级不同。直接相加相当于隐式地假设两者同等重要，但实际上某一方可能数值远大于另一方，导致梯度被该任务主导，另一个任务得不到充分训练。

### 坏处二：任务难度不均衡

意图识别类别少（比如本项目只有24类），收敛快；实体识别标签多（BIO展开后有百来类），收敛慢。直接相加的情况下，已经收敛的任务的梯度仍然在干扰共享的 BERT 参数，对慢收敛的任务造成负面影响。

### 坏处三：梯度可能冲突

两个任务对共享 BERT 参数的梯度方向不一定一致，甚至可能相反。直接加法无法解决梯度冲突，结果是两个任务都训练得不够好，不如单独训练某一个任务的效果。

### 坏处四：权重是固定的超参

直接相加隐含了两个任务权重都是1，实际上最优权重因数据集、任务难度不同而变化，需要人工反复调试，成本高且不鲁棒。

## 三、存在训练不平衡时的处理方法

### 方法一：手动加权求和

根据两个任务在验证集上的表现，手动给表现差的任务赋予更高权重，让模型更多关注该任务。缺点是需要反复实验调参，不够自动化。

### 方法二：自动学习任务权重（不确定性加权）

引入可学习的参数来表示每个任务的不确定性，让模型在训练过程中自动平衡两个任务的贡献比例，不需要人工设定权重，是比较优雅的解决方案。

### 方法三：分阶段训练

先单独训练容易收敛的任务（意图识别）使其稳定，再加入难收敛的任务（实体识别）进行联合训练。或者先预热训练某个任务，再逐步引入另一个任务，避免两个任务在早期互相干扰。

### 方法四：监控 loss 比值动态调整

在训练过程中监控两个任务的 loss 变化速度，对下降慢的任务动态增大权重，对已经收敛的任务动态减小权重，让两个任务保持相近的学习进度。
