1-1：
文本分类：利用 BERT 输出的 [CLS] 向量（pooler_output），经过一个全连接层得到意图类别。它捕捉整个句子的语义，用于判断全局意图。

实体识别（槽位填充）：利用 BERT 输出的每个 token 的向量（token_output），经过另一个全连接层得到每个 token 的标签（B/I/O）。
它关注局部信息，用于识别句子中的具体实体。

两者的关系是：意图和实体往往相互关联，共享底层表示可以互相促进（例如知道了“查天气”的意图，更容易识别出“地点”实体）。

文本分类：使用 nn.CrossEntropyLoss，计算预测的意图类别与真实意图之间的交叉熵。

实体识别：也使用 nn.CrossEntropyLoss，但只对有效 token（即 attention_mask 为 1 的位置）计算损失。
代码中通过 active_loss 和 active_logits 正确实现了这一点。

1-2：
直接相加 loss = seq_loss + token_loss 的坏处
直接将两个 loss 相加，隐含地假设了两个任务的重要性相同且量纲一致。但实际情况中：

量纲差异：seq_loss 是单个标量（一个句子一个意图），而 token_loss 是对多个 token 求和/平均后的值，其数值范围可能远大于 seq_loss。
这会导致模型更关注实体识别任务，而忽视意图分类。

收敛速度不一致：两个任务的学习难度可能不同，一个任务可能很快收敛，另一个却仍在波动，直接相加可能让梯度相互干扰。

梯度冲突：共享的 BERT 层需要同时满足两个任务的目标，如果两个任务的梯度方向不一致，可能导致参数更新不稳定，甚至损害性能。

处理任务不平衡的方法:
加权求和：引入超参数 alpha 和 beta，例如 loss = alpha * seq_loss + beta * token_loss。
可以通过实验调整，或者根据 loss 的初始值动态设置（例如让两者初始量级相近）。

动态权重调整：使用像 GradNorm 或 Uncertainty Weighting 的方法，让模型自动学习每个任务的权重

任务特定的学习率：可以为两个任务的分类头设置不同的学习率，或者对共享的 BERT 层和任务头使用不同的学习率。

交替训练：每个 batch 或每个 epoch 轮流优化一个任务，但这种方法在实践中较少使用。

检查数据平衡：确保两个任务的数据量没有严重偏差，如果实体标签很多而意图类别很少，可以适当增加意图分类的权重
