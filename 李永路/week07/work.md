### 问题 1: BERT 文本分类和实体识别有什么关系，分别使用什么 loss？

#### 关系说明

在本项目中，BERT 同时执行**意图识别（文本分类）**和**槽位填充（实体识别/序列标注）**两个任务，采用**多任务联合学习**架构。

### 问题 2: 多任务训练 `loss = seq_loss + token_loss` 有什么坏处？如果存在训练不平衡的情况，如何处理？
#### 直接相加的弊端
1. **量纲和数值范围不匹配**
2. **收敛速度不一致**
3. **梯度冲突问题**
4. **数据分布不平衡**
5. **任务优先级无法体现**

#### 处理训练不平衡的方法
- 方法 1: 手动加权求和
- 方法 2: 不确定性加权
- 方法 3: 按样本数归一化
- 方法 4: 梯度裁剪与归一化
- 方法 5: 动态权重调整



你是一个专业的交通出行领域语义解析专家。请分析用户的自然语言查询，提取结构化信息。

【任务要求】
1. 识别用户意图（intent）：从以下类别选择
   - QUERY（查询）、BOOK（预订）、CANCEL（取消）、COMPARE（对比）
   
2. 识别领域（domain）：从以下类别选择
   - train（火车）、flight（飞机）、bus（公交）、map（地图）等
   
3. 抽取实体槽位（slots）：识别并标注以下类型的实体
   - Src（出发地）、Dest（目的地）、date（日期）、time（时间）
   - transportation_type（交通工具类型）、seat_class（座位等级）等

【输出格式】
请严格按照以下 JSON 格式输出：
{
    "domain": "领域标签",
    "intent": "意图标签", 
    "slots": {
        "实体类型 1": "实体值 1",
        "实体类型 2": "实体值 2"
    }
}

【示例】
输入："查询许昌到中山的高铁票"
输出：
{
    "domain": "train",
    "intent": "QUERY",
    "slots": {
        "Src": "许昌",
        "Dest": "中山",
        "transportation_type": "高铁"
    }
}
