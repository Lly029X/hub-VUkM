### BERT 文本分类和实体识别有什么关系，分别使用什么 loss？
关系：
- 从 model.py 可以看出，这是一个**多任务学习（Multi-task Learning）**架构：
    - 共享编码器：两个任务共用同一个 BERT 预训练模型作为特征提取器
    - BERT 输出的 [CLS] 位置向量（pooler_output）用于句子级别的意图分类
- BERT 输出的每个 token 的隐藏状态（token_output）用于 token 级别的实体识别
- 不同的分类头：
    - 意图识别（句子级）：使用 [CLS] 向量 → Dropout → Linear(768 → 意图个数)
    - 实体识别（token 级）：使用每个 token 向量 → Dropout → Linear(768 → 实体类别个数)
- 相互促进：
    - 意图信息有助于实体识别（例如知道是"查询车票"意图，更容易识别出发地、目的地实体）
    - 实体信息也有助于意图判断（出现地名实体可能是交通查询意图）

### 多任务训练 loss = seq_loss + token_loss 有什么坏处？如果存在训练不平衡，如何处理？

- 梯度量级不平衡：
    - Token 级别的任务样本数远多于句子级别（一句话有几十个 token，但只有 1 个句子标签）
    - 导致 token_loss 的梯度可能远大于 seq_loss，训练被实体识别任务主导
- 收敛速度不同：
    - 简单任务可能很快收敛，困难任务还在缓慢学习
    - 先收敛的任务梯度变小，另一个任务主导训练
- 负迁移（Negative Transfer）：
    - 如果两个任务相关性不强，可能出现互相干扰
    - 一个任务的优化可能损害另一个任务的性能
- 权重难以调优：
    - 简单的 1:1 相加可能不是最优的
    - 需要大量实验来调整权重比例
